# 사용자 시나리오 (User Scenarios)

이 문서는 '마이링크' 서비스를 이용하는 두 가지 주요 페르소나(방문자, 소유자)의 행동 흐름을 정의합니다. 모든 시나리오는 "사용자는 ___하기 위해 ___한다" 형식을 따릅니다.

## 1. 방문자 (Visitor): 탐색 및 정보 확인
사용자가 타인의 마이링크 페이지에 접속했을 때의 경험입니다.

*   **페이지 접속:** 사용자는 상대방의 통합된 프로필을 확인하기 위해 공유된 URL(`mylink.com/{slug}`)로 접속한다.
*   **전문성 파악:** 사용자는 상대방의 주요 기술 역량을 한눈에 보기 위해 상단에 노출된 기술 스택 배지를 확인한다.
*   **외부 서비스 이동:** 사용자는 상대방이 운영하는 블로그, SNS, 포트폴리오 등으로 이동하기 위해 각 링크 카드를 클릭한다.
*   **모바일 최적화 경험:** 사용자는 이동 중에도 불편함 없이 정보를 확인하기 위해 스마트폰 환경에 최적화된 레이아웃을 이용한다.

## 2. 소유자 (Owner): 프로필 및 콘텐츠 관리
사용자가 자신의 페이지를 설정하고 정보를 업데이트하는 경험입니다.

### 2.1. 인증 및 초기 설정
*   **초기 가입 및 생성:** 사용자는 나만의 디지털 명함을 빠르게 만들기 위해 구글 계정을 연동하여 자동 가입한다.
*   **고유 주소 설정:** 사용자는 기억하기 쉬운 나만의 주소를 갖기 위해 닉네임을 수정하고 이와 동기화되는 URL 슬러그를 설정한다.

### 2.2. 프로필 및 링크 관리
*   **프로필 브랜딩:** 사용자는 방문자에게 신뢰감을 주기 위해 프로필 이미지, Bio(소개글), 전문 분야(Tech Stack)를 편집한다.
*   **링크 추가:** 사용자는 새로운 프로젝트나 홍보 채널을 알리기 위해 URL을 입력하여 링크 카드를 생성한다.
*   **시각적 자동 완성:** 사용자는 링크의 가독성을 높이기 위해 URL 입력 시 자동으로 연동되는 파비콘 아이콘을 확인한다.
*   **인라인 편집:** 사용자는 별도의 관리 페이지를 거치지 않고 조회 화면에서 즉시 닉네임이나 소개글을 수정하여 변경 내용을 실시간으로 반영한다.
*   **링크 최신화/정리:** 사용자는 오래된 정보를 삭제하거나 잘못 입력된 URL을 수정하여 프로필의 정확성을 유지한다.

### 2.3. 고급 설정 및 레이아웃
*   **레이아웃 최적화:** 사용자는 강조하고 싶은 프로젝트를 돋보이게 하기 위해 Bento(그리드) 스타일의 레이아웃을 적용한다.

## 3. 추후 확장 시나리오 (Backlog)
현재 단계에서는 포함되지 않으나, 향후 고도화 시나리오입니다.

*   **공유 및 확산:** 사용자는 자신의 마이링크를 널리 알리기 위해 페이지 내 공유 버튼을 눌러 링크를 복사하거나 SNS로 전송한다.
*   **성과 측정:** 사용자는 어떤 링크가 가장 인기가 많은지 파악하기 위해 각 링크의 클릭 수 통계를 확인한다.
*   **순서 변경:** 사용자는 중요한 링크를 상단에 배치하기 위해 드래그 앤 드롭으로 링크 카드의 순서를 변경한다.
